01 / Logo System

The Mark & Wordmark

The Store Kholo logo combines a geometric mark with a structured wordmark. The mark is a modular grid symbol — two letterforms (S + K) fused into a system diagram — referencing the connected, architectural nature of what we build.

Logo Anatomy
01 /
The Mark
A connected-node system mark on a strict orthogonal grid. Three off-white nodes form an L-shaped structural axis — the foundation layer. Two red nodes branch off mid-axis — the active system layer. No letterforms. Reads cleanly from 16px to 128px.
02 /
The Wordmark
"Store Kholo" set in Sora ExtraBold, all caps optically adjusted. Letter-spacing at +0.06em. No ligatures. The name sits on the baseline of commerce — direct, readable, confident. Never italicized.
03 /
The Tagline
"Commerce Systems · Built to Scale" in IBM Plex Mono at 0.48rem, +0.28em tracking. Appears beneath the wordmark only in full-lockup usage. Rendered in muted tone — it supports, never competes.
04 /
Clear Zone
Minimum clear zone equals the height of the red corner square on all four sides. No graphic elements, text, or imagery may encroach this space. On digital, minimum logo width is 120px for the full lockup, 32px for the mark alone.

04 / Visual Language

How the Brand Behaves

Visual language is everything below the logo — the patterns, motions, textures, and spatial logic that make the brand feel consistent across every surface.

Grid System
8pt Base Grid
All spacing, sizing, and layout decisions live on a strict 8pt grid. 12-column desktop, 4-column mobile. Gutters: 24px. Margins: 48px desktop, 24px mobile.
Spacing Scale
Deliberate Density
Generous vertical space between sections creates breathing room. Dense information within components signals precision. Tension between the two creates visual authority.
Motion Principles
Purposeful Only
Animations reveal structure, never decorate. Entry: fade + translateY(24px), 0.6s ease. Hover: opacity shift + 1px Y-translate. No bounces. No spins. No attention-seeking motion.
Border Language
Structural Lines
Borders define space, not style. Rules use rgba(255,255,255,0.07). Accent borders signal activity or hierarchy. The red left-border on active states is a universal system signal.
Active state border
Iconography
Geometric & Square
Icons use a 24×24 grid, 1.5px stroke, square linecaps. No rounded corners. No filled icons. Monochrome — off-white on dark, or red for active/alert states.

06 / Usage Rules

Do & Don't

Brand consistency is enforced by clear rules, not good intentions. These apply to every touchpoint.

✓ DO
Use Signal Red for one dominant element per visual field — a CTA, an accent line, or a key data value.
Set headings in Sora with negative tracking (-0.02em or tighter).
Keep the logo on dark, light, or red backgrounds only — never on photography directly.
Use IBM Plex Mono exclusively for technical labels, data values, and system indicators.
Maintain the grain texture on all full-bleed dark sections.
Use the section label pattern (line + mono text + number) to open every new content section.
✗ DON'T
Never use red as a background fill for content areas — only for the logo field and CTA buttons.
Never use gradients as primary design elements. One subtle radial gradient max per section.
Never italicize the logo wordmark or distort the mark's proportions.
Never use Sora for body copy or IBM Plex Mono for headings — roles are fixed.
Never place the full logo below 120px wide on digital, or 20mm on print.
Never add drop shadows to the logo mark. The mark is flat and structural — always.
